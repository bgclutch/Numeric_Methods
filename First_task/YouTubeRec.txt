Тема: Рекомендательные системы видеохостинга YouTube
1. Постановка решаемой задачи
Необходимо в реальном времени подбирать для каждого из 2.5+ миллиардов
пользователей персонализированный список видео, которые они с наибольшей
вероятностью захотят посмотреть.
Цель: Максимизация времени просмотра и удержания пользователей на платформе.
Сложность: Выборка лучших ~10 видео из миллиардов доступных вариантов c
задержкой менее 200 мс.
2. Размерности данных
Общий архив сотни экзабайт (10 18 байт), более 500 часов видео загружается каждую
минуту, около 2.5 млрд активных пользователей в месяц. Триллионы лайков и просмотров
ежедневно. Каждое видео и пользователь представляются векторами высокой размерности
для поиска похожих.
3. Используемые вычислительные методы
Используется двухуровневая архитектура глубоких нейронных сетей:
1. Генерация кандидатов: алгоритм быстро отсеивает из миллиардов видео несколько
сотен потенциально интересных (метод Approximate Nearest Neighbors).
2. Ранжирование: более тяжелая нейросеть детально оценивает эти сотни видео,
присваивая каждому «вес» полезности, учитывая контекст, историю и свежесть
контента.
Также используется Reinforcement Learning (обучение с подкреплением) для
оптимизации долгосрочного удовлетворения пользователя, а не просто кликов.
3. Используемое программное и аппаратное обеспечение
Аппаратное обеспечение:
Google TPU (Tensor Processing Unit): специализированные ASIC-чипы для
ускорения матричных вычислений в нейросетях.
Распределенные кластеры CPU/GPU в дата-центрах Google по всему миру.
Программное обеспечение:
TensorFlow: Фреймворк для машинного обучения.
BigTable / MapReduce: Для обработки и хранения логов активности.
Контейнеризация (Borg/Kubernetes) для управления нагрузкой.
5. Полученные результаты
Эффективность: По статистике, более 70% времени, которое люди проводят на
YouTube - это просмотр видео, предложенных рекомендательным алгоритмом, а не
найденных через поиск.
Скорость: Система способна обновлять рекомендации в реальном времени сразу
после того, как пользователь нажал на новое видео.

Масштабируемость: Успешная обработка запросов от миллиардов устройств
одновременно без падения сервиса.
